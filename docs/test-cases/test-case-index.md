## 测试执行建议

### 冒烟测试 (每日执行)
建议选择以下关键测试用例进行每日冒烟测试：
- AUTH-001, AUTH-016 (登录登出)
- USER-001, USER-007 (用户管理基本功能)
- NEWS-001, NEWS-007 (新闻管理基本功能)
- COLLECT-001, COLLECT-011 (新闻采集基本功能)
- PUBLISH-001, PUBLISH-004 (新闻发布基本功能)
- DASHBOARD-001 (仪表板基本功能)

### 回归测试 (版本发布前)
- 执行所有高优先级测试用例
- 执行与修改功能相关的中优先级测试用例
- 执行核心业务流程的端到端测试

### 全量测试 (重大版本发布)
- 执行所有测试用例
- 包含性能测试、安全测试、兼容性测试
- 进行压力测试和稳定性测试
